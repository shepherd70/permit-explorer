- 2026-06-21 — Reworked the city map: removed the "Bubbles" display option so it is now ALWAYS a community choropleth, and added "Avg permits / year" as a new shading metric and the default (PR session/2026-06-21-permit-category-filter). `ppy` = a community's permit count ÷ the selected year-range span (log-scaled) — the volume signal the proportional-count bubbles used to carry, so the default map still answers "which communities are building the most?". Removed `drawBubbles()`, `setChoro()`, the bubbles/areas segmented control + its CSS, the `this.choro` state, and the `map=areas` URL param (legacy `?map=areas` links are now ignored; the `metric=` param omits the new default `ppy`). - 2026-06-21 — Added a permit-category show/hide filter to the city-wide explorer (`src/city_explorer.html`; PR session/2026-06-21-permit-category-filter). A multi-select checkbox panel — built dynamically from the distinct `permitclassgroup` values (Single Family, Commercial, Two Family, Garage, …) — replaces the old single-select "Permit class" dropdown. Every category starts checked except **Single Family** (≈55% of permits; clutters the map), which starts unchecked so the default view excludes it but leaves it one click away (if a data pull has no Single Family, everything starts checked). The filter is server-side (`permitclassgroup IN (<checked set>)` added to `where()`, omitted when all are checked, defensive `IN ('')` when none), so the count query — and therefore the city/detail ~30,000 threshold — evaluates the FILTERED count: hiding Single Family can flip a scope into detail view. Toggling re-runs the existing AbortController/`_seq`-guarded `apply()` (a refetch — city mode has no client-side rows to filter); clearing every box short-circuits to an empty state with no network call; a "Select all / Clear all" control and an "N of M shown" count round it out. View state is shareable via `?cats=` (encodes the HIDDEN set; default omitted for clean links, `cats=*` = show all; self-heals on load). Kept the single-`<script>` harness contract and guarded every browser-only call; `esc()` left untouched so `drift.js` stays green (the one new attribute sink escapes its own `"` inline). City harness extended 40→59 — including a category-aware count stub plus a city→detail flip assertion that guards the "threshold on the filtered count" invariant (verified it hard-fails if the `IN` clause is dropped from the count query).
